1. All Equations Used in the Project

📐 Equation 1 — Eye Aspect Ratio (EAR) for Blink Detection

File: blink_api/app.py

Used to detect eye blinks from video frames. MediaPipe extracts 6 landmark points per eye:

EAR = (A + B) / (2.0 × C)

Where: - A = ‖P₂ − P₆‖ (vertical distance between upper-left and lower-left eyelid) - B = ‖P₃ − P₅‖ (vertical distance between upper-right and lower-right eyelid) - C = ‖P₁ − P₄‖ (horizontal distance between eye corners) - ‖ ‖ = Euclidean norm: √((x₂−x₁)² + (y₂−y₁)²)

⚠️ IMPORTANT:
Threshold: EAR < 0.21 → Eye is closed
Blink counted when EAR stays below threshold for ≥ 2 consecutive frames

The average EAR of both eyes is used:

EAR_avg = (EAR_left + EAR_right) / 2.0

📐 Equation 2 — Blink Rate (Blinks Per Minute)

File: CameraScanScreen.kt

ML Kit detects blinks in a 15-second window, then extrapolates to per-minute rate:

Blink_Rate_BPM = blink_count_in_15s × 4

ℹ️ NOTE:
Eye closed threshold: eyeOpenProbability < 0.2
Eye open threshold: eyeOpenProbability > 0.5

📐 Equation 3 — Accelerometer Magnitude (Step Detection & Motion)

File: StepManager.kt and MotionSensorManager.kt

magnitude = √(x² + y² + z²)

Where x, y, z are the 3-axis accelerometer readings.

For step detection (StepManager):

IF magnitude > 11.5 AND lastMagnitude ≤ 11.5 AND (currentTime − lastStepTime) > 300ms
THEN step_count++

For motion/stillness detection (MotionSensorManager):

deviation = |magnitude − 9.81|

IF deviation > 0.5 → motion detected (not still)
IF deviation ≤ 0.5 for ≥ 20 minutes → declared "still"

📐 Equation 4 — Heart Rate Estimation (Peak Detection via PPG)

File: CameraScanScreen.kt

Uses photoplethysmography (PPG) — finger over rear camera with flashlight:

Step 1: avg_brightness = Σ(pixel_values) / num_pixels        (per frame)
Step 2: smoothed = moving_average(signal, window=5)
Step 3: mean = average(smoothed)
Step 4: peaks = count(transitions from below_mean → above_mean)
Step 5: duration_seconds = num_frames / 30.0                  (assuming 30 fps)
Step 6: BPM = (peaks / duration_seconds) × 60

Final HR uses median of last 15 readings for robustness:

sorted_readings = sort(heartRateBuffer)
IF even count: median = (sorted[n/2 - 1] + sorted[n/2]) / 2
IF odd count:  median = sorted[n/2]

📐 Equation 5 — Sleep Confidence Score (Weighted Multi-Signal)

File: SleepDetectionManager.kt

Confidence = W_screen + W_motion + W_window + W_no_unlock
Signal Weight Full Score Condition 2/3 Score 1/3 Score
Screen Off 30 ≥ 30 min off ≥ 15 min ≥ 10 min
Motion Still 30 ≥ 30 min still ≥ 20 min ≥ 10 min
Sleep Window 25 Currently in predicted window — —
No Unlock 15 Last unlock > 15 min ago — —

Penalty: If not still but screen off and motion magnitude > 1.0 → score −10

Final Confidence = clamp(score, 0, 100)

State transitions: - AWAKE → POSSIBLY_SLEEPING: confidence ≥ 40 + screen off ≥ 15 min - POSSIBLY_SLEEPING → SLEEPING: confidence ≥ 60 + screen off ≥ 30 min + still

📐 Equation 6 — Circular Mean for Sleep Window Prediction

File: SleepPatternRepository.kt

Handles midnight wraparound (e.g., averaging 23:00 and 01:00):

For each hour h:
    θ = h × 2π / 24

sinSum = Σ sin(θᵢ) / n
cosSum = Σ cos(θᵢ) / n

mean_angle = atan2(sinSum, cosSum)
IF mean_angle < 0: mean_angle += 2π

mean_hour = mean_angle × 24 / (2π)

📐 Equation 7 — Sleep Duration Calculation

File: SleepDetectionManager.kt

duration_ms = wakeTime − sleepStartTime
duration_hours = duration_ms / (1000 × 60 × 60)

Clamped to range [0.0, 16.0] hours. Only saved if ≥ 1.0 hour.

📐 Equation 8 — Estimated Steps (Fallback)

File: HealthConnectManager.kt

waking_hours = current_hour − 7       (for hours 7-22)
estimated_steps = waking_hours × 350

📐 Equation 9 — Estimated Sleep from Time (Last Resort Fallback)

File: HealthConnectManager.kt

IF hour in [5..11]:  estimated_sleep = clamp(hour − 23 + 24, 5.0, 8.0)
IF hour in [12..23]: estimated_sleep = 7.0
ELSE:                estimated_sleep = 4.0

📐 Equation 10 — ML Model: StandardScaler Transform

File: retrain_model.py

X_scaled = (X − μ) / σ

Where μ and σ are per-feature mean and standard deviation from training data.

📐 Equation 11 — Stress Level from Score (Dashboard Fallback)

File: DashboardViewModel.kt

IF stress_score < 40 → "Low"
IF stress_score < 70 → "Mid"  
ELSE                 → "High"

7. Sleep Detection State Machine

stateDiagram-v2
    [*] --> AWAKE
    AWAKE --> POSSIBLY_SLEEPING : Screen off >= 15min AND confidence >= 40
    POSSIBLY_SLEEPING --> SLEEPING : Screen off >= 30min AND still AND confidence >= 60
    POSSIBLY_SLEEPING --> AWAKE : Screen on and not brief unlock
    SLEEPING --> AWAKE : Real unlock and calculates duration
    SLEEPING --> SLEEPING : Brief unlock ignored

5. Health Connect API

Sleep:

1. SleepDetectionManager (real-time multi-signal engine)
